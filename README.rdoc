== Дано
Чат, где общаются родитель и ситтер. Реализовать функцию, которая будет решать нужно ли отправить смс-уведомление пользователю о новом сообщении для него.
Функция вызывается не сразу, а через некоторое время после каждого сообщения и решает нужно ли отправить смс об этом конкретном сообщении. За это время могут быть добавлены новые сообщения.
Смс не надо отправлять после каждого сообщения, чтобы избежать лишних раcходов. Т.е. смс должны уходить только после первого непрочитанного сообщения в цепочке сообщений от одного отправителя.

== Правила такие:
* отправлять уведомление о сообщении, если сообщение еще не прочитано.
* не отправлять уведомление, если есть предыдущие непрочитанные сообщения от этого отправителя
Решение должно быть с тестами. Необязательно использовать предложенный код со стукторой данных. Он дан для примера. Можно использовать любые фреймоворки и библиотеки, подключать БД, хранить данные там и использовать SQL для решения.
